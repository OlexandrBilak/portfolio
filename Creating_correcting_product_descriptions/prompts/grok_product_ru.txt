Цель
Создать профессиональное, структурированное и практическое описание товара на основе исходных характеристик. Текст помогает покупателю принять рациональное решение, полностью опираясь на характеристики, и превосходит типовые карточки за счёт глубины объяснения характеристик, без пафоса и маркетинговых клише.
 
Основные принципы
1.	Строго по характеристикам
•	Использовать 100% технических характеристик из исходного описания.
•	Запрещено:
o	выдумывать факты, функции, цифры;
o	добавлять сценарии использования, не вытекающие из характеристик;
o	использовать маркетинговые клише («лучший выбор», «высокое качество», «оптимальное решение»).
2.	Характеристика → польза
•	Каждая характеристика раскрывается один раз в блоке, где она логично объясняет практическую пользу.
•	Обязательно раскрывать:
o	зачем характеристика нужна;
o	кому она полезна;
o	какие задачи решает.
•	Не дублировать функции, цифры и данные в разных блоках.
3.	Фокус на практическую пользу
•	Формулировки строго привязаны к характеристикам и их реальной пользе.
•	Запрещены абстрактные эпитеты, эмоции и художественные сравнения.
4.	Стиль и структура предложений
•	Профессиональный, коммерческий, от лица специалиста.
•	Средняя длина предложений: 8–18 слов.
•	Сложные конструкции делить на 2 предложения.
•	Без повторов и «AI воды».
5.	Логика текста
•	Последовательная структура, отсутствие противоречий.
•	Не вставлять информацию, отсутствующую в исходном описании.
6.	SEO и ключевые слова
•	Ключевые слова использовать только логично; главный ключ — название товара и естественные вариации.
•	Главный ключ должен быть в первых 200–300 символах вступления.
•	Коммерческие слова («купить», «цена») — только при естественной необходимости.
7.	Упоминание компании (опционально, ненавязчиво)
•	Компания может быть упомянута кратко:
o	работает с 2010 года;
o	консультирует честно;
o	предлагает проверенные решения;
o	гарантия от производителя;
o	адекватные цены, возможны скидки;
o	широкий ассортимент.
•	Формулировки должны быть разными между товарами.
8.	Призыв к консультации
•	Короткий, нейтральный, без давления:
«Якщо не впевнені у виборі або потрібна оптимальна конфігурація — напишіть або зателефонуйте, ми допоможемо підібрати рішення.»
 
Обязательная структура текста
1.	Вступление (2–3 предложения)
•	Главная выгода товара в первых 200–300 символах.
•	Тип товара, назначение, целевая аудитория.
•	Только факты из характеристик.
2.	Ключевые преимущества и особенности модели (4–7 пунктов)
•	Каждый пункт строго на основе характеристик.
•	Каждое преимущество раскрывает практическую пользу.
•	Формат: список или короткие абзацы.
•	Не повторять данные из других блоков.
3.	Где и для кого применяется товар
•	Логично вытекает из характеристик.
•	Указать типовые решаемые задачи.
•	Без вымышленных сценариев.
4.	Для кого товар НЕ подходит (1–3 предложения)
•	Реальные ограничения, вытекающие из характеристик.
•	Без категоричности и негативной окраски.
5.	Характеристики → польза (ключевой блок)
•	Не используй характеристики, которые рассматривались ранее.
•	Только важные параметры для выбора.
•	Каждая характеристика раскрывается через практическую пользу.
•	Не повторять характеристики из блока преимуществ.
•	Без «технических простыней».
6.	Сравнение с категорией (3–5 предложений)
•	Без конкретных моделей.
•	Подчеркнуть сильные стороны данной модели или категории.
•	Кратко, только фактические отличия.
7.	Мягкий призыв к консультации
•	Шаблонный, нейтральный.
 
HTML-оформление
•	Абзацы: <p>
•	Разделы: <h2>, <h3>
•	Списки: <ul><li> (1–2 списка при необходимости)
•	Выделение: <strong> — только для ключевых коммерческих или навигационных акцентов
•	Таблицы — только если они лучше передают суть, без указания размеров
•	Изображения сохранять, добавлять alt текст
 
Метаданные
1.	Название товара (для магазина): тип + бренд + модель + ключевая особенность
2.	HTML title: до 80 символов, начинается с названия, допускаются «купить», «цена»
3.	META description: до 170 символов, кратко о пользе и преимуществах, упоминание консультации, 1 эмодзи допустим
 
Контроль качества (валидация)
•	Текст выглядит как человеческое описание.
•	Логическая структура, читабельность, отсутствие повторов и шаблонов.
•	Приоритет: практическая польза и помощь в выборе, форма вторична.
•	Объём: ориентировочно 150–400 слов (допускается больше для полного раскрытия характеристик).
